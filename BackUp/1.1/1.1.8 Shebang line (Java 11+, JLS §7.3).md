### Shebang Line (Java 11+)

Начиная с Java 11, исходный файл может использоваться как исполняемый скрипт.  
Если первая строка файла начинается с символов `#!` (`shebang`), компилятор игнорирует её как часть исходного кода.

Пример:
```java
#!/usr/bin/java --source 11
public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```
- `Shebang` должен быть строго первой строкой файла.
- Обычно применяется вместе с режимом запуска исходников (`java File.java`).
- В остальном правила анализа кода остаются прежними.

Замечания
- `Shebang` поддерживается только в `Unix`-подобных системах (`Linux`, `macOS`), где оболочка использует первую строку для выбора интерпретатора. На `Windows` такая строка игнорируется, и запуск файла как скрипта напрямую невозможен.
- Обычно в `shebang` указывают параметр `--source <версия>` (например, `--source 11`), чтобы гарантировать совместимость кода с нужной версией языка. Если параметр не указан, используется версия по умолчанию, зависящая от установленной `JVM`.
- Перед строкой `shebang` не допускаются комментарии или пробельные символы. Она должна строго начинаться с первого символа файла.
